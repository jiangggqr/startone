00 总览与完整流程

产品信息架构、学习闭环、Agent 边界和生产级设计要求。

StartOne · Product Blueprint上传材料是唯一来源 · 一次一个下一步
一句话定义用户上传自己的学习材料，系统把材料转化为可核查的知识地图和短学习任务；Tutor 负责当前知识点的讲解与练习，规划 Agent 根据学习证据只选择一个下一步。
上传材料→可视化知识框架→单概念理解→主动提取→真实小胜利→自动进入下一任务→总结与恢复

Guided Mastery Loop

Tutor、Quiz、复述、提示、反馈、鼓励、补救练习都围绕当前概念工作，并产生客观学习证据。

不改变全局路线，不引入上传材料之外的事实。

Adaptive Planning Agent

读取证据后只选择一个动作：继续、重试、换练习、简化、插入前置知识、复习、请求补充材料或结束。

安全动作自动执行；材料缺口只打开上传区并保留当前概念。

来源可信

每个概念、题目和解释都带来源标签和可打开的位置。

ADHD-informed

一个概念、一个主动作、可暂停、自动保存、低威胁反馈。

生产级状态

加载、空、错误、部分成功、离线、模型失败均有恢复路径。

无障碍与响应式

键盘可用、焦点清晰、语义正确、桌面至手机均可完成核心流程。

实现顺序来源与状态 → 知识地图 → 专注学习界面 → Tutor → Quiz/复述/反馈 → Agent → 材料缺口恢复 → 全流程测试。

键盘与焦点

Tab 顺序符合视觉顺序；模态框内焦点受控；关闭后焦点返回触发按钮。

所有主要操作都有可见焦点；不依赖悬停；不能只用颜色传达正确与错误。

目标尺寸

规范最低遵循 WCAG 2.2 的 24×24 CSS px；本产品主按钮和常用触控目标默认至少 44×44。
