00 总览与完整流程

产品信息架构、学习闭环、Agent 边界和生产级设计要求。

StartFrame Agent · Product Blueprint上传材料优先 · 搜索受控 · 一次一个下一步
一句话定义用户上传自己的学习材料，系统把材料转化为可核查的知识地图和短学习任务；Tutor 负责当前知识点的讲解与练习，规划 Agent 根据学习证据只选择一个下一步。
上传材料→AI 选择学习重点→确认知识地图→2 分钟基础信号→概念解释→Tutor / Quiz / 复述→反馈与补救→LearningEvidence→Agent 下一步→总结与恢复

Guided Mastery Loop

Tutor、Quiz、复述、提示、反馈、鼓励、补救练习都围绕当前概念工作，并产生客观学习证据。

不改变全局路线，不直接启动网络搜索。

Adaptive Planning Agent

读取证据后只选择一个动作：继续、重试、换练习、简化、插入前置知识、复习、请求搜索或结束。

必须显示面向用户的理由，并允许用户选择其他路径。

来源可信

每个概念、题目和解释都带来源标签和可打开的位置。

ADHD-informed

一个概念、一个主动作、可暂停、自动保存、低威胁反馈。

生产级状态

加载、空、错误、部分成功、离线、模型失败均有恢复路径。

无障碍与响应式

键盘可用、焦点清晰、语义正确、桌面至手机均可完成核心流程。

实现顺序来源与状态 → 知识地图 → 专注学习界面 → Tutor → Quiz/复述/反馈 → Agent → 受控搜索 → 全流程测试。

你已经掌握

你说出了两个核心动作：判断当前位置与其他位置的关系，以及把重要信息加入当前位置。

需要补充或澄清

简练修正答案

具体鼓励

你已经抓住了“先判断关系，再加入信息”的结构。现在只需要把“怎样加入”说得更准确。
