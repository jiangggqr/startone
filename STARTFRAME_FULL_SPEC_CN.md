# StartFrame Agent 完整产品方案

## 1. 产品定义

StartFrame Agent 是一个以上传学习材料为核心、由适应性学习规划 Agent 驱动，并包含情境化 Tutor 的 Web App。

它不是：

- 自动抓取大量链接的搜索产品
- 只有聊天框的通用 AI Tutor
- 只有番茄钟、待办或选择题的学习工具
- 声称诊断或治疗 ADHD 的医疗产品

它解决的核心问题是：

> 用户已经有材料和学习目标，但难以开始、难以判断先学什么，也难以在不懂时得到连续、低压力且有来源的帮助。

## 2. 目标用户

首要用户是使用课程讲义、论文、教材章节和笔记进行自学的大学生与成人学习者，尤其关注可能存在任务启动困难、工作记忆负担、时间感不清和错误反馈压力的用户。

产品使用 ADHD-informed 设计，但不要求用户声明诊断，也不把所有学习困难归因于 ADHD。

## 3. 内容来源优先级

### 第一优先：用户上传材料

PDF、Markdown、TXT 或粘贴文本。系统保留真实页码、标题和行号。

### 第二优先：经用户确认的外部补充

只有“四重搜索门”同时满足时搜索：

1. 会话设置允许系统建议搜索。
2. 存在明确、命名并经过服务端验证的 `SourceGap`。
3. Agent 选择 `request_search`。
4. 用户在本次搜索前确认目标与范围。

### 第三优先：AI 补充解释

用于简化语言、定义术语或生成例子。界面必须标记为“AI 补充解释”，不能伪装成原文。

## 4. 五个系统

### 4.1 学习内容与来源系统

负责上传、解析、真实定位、检索、来源预览、来源覆盖和缺口识别。

不负责教学、评分或决定下一步。

### 4.2 ADHD-informed 专注会话

负责两分钟启动、一屏一个概念、一个主动作、可选时间显示、自动保存、暂停、恢复和下次启动动作。

不负责判断答案。

### 4.3 Guided Mastery Loop

统一负责当前概念的：

- Tutor 对话
- 连续引导
- Quiz
- 自由复述
- 分层提示
- 即时反馈
- 具体鼓励
- 错误驱动的补救练习
- `LearningEvidence` 生成

### 4.4 Adaptive Planning Agent

学习适应判断只读取经过验证的学习证据；同时读取路线、时间、精力、前置关系、来源覆盖和重试次数作为约束，只选择一个全局下一动作：

- `continue_next`
- `retry_current`
- `switch_activity`
- `simplify_current`
- `insert_prerequisite`
- `review_previous`
- `request_search`
- `finish_session`

Agent 不负责讲解、出题、评分或写鼓励文案。

Guided Mastery Loop 可以在 Feedback 中给出一个局部 `next_micro_action`，但它只能继续当前概念内的练习，不能改变路线、搜索或结束会话。

### 4.5 平台、状态与信任层

负责 FastAPI、SQLite、GPT-5.6 Responses API、结构化输出、工具执行、状态机、权限、自动保存、Demo mode、错误恢复、测试和部署。

## 5. 完整用户旅程

1. 首页以“上传材料并新建学习会话”为主入口，同时显示历史会话。
2. 输入目标、已有基础、时间、精力、语言和支持偏好。
3. 上传材料并查看每个文件的解析状态；用户也可从首页上传入口先选文件，再补充会话设置。
4. 审阅材料覆盖、真实来源位置和候选缺口。
5. Agent 生成知识地图和本次路线，用户确认或调整。
6. 系统只显示一个 60–120 秒启动动作。
7. 进入专注工作区，一次学习一个概念。
8. 用户可以打开 Tutor、Quiz、复述、提示和来源查看器。
9. 系统立即给出结构化反馈、具体鼓励和必要补救练习。
10. 所有活动形成客观 `LearningEvidence`。
11. Agent 显示一个推荐下一步、理由和预计时间，用户可选择其他路径。
12. 只有四重搜索门全部满足时才执行外部搜索。
13. 会话结束时保存总结、误区、进度和下次两分钟启动动作。
14. 用户可在学习记录中恢复、复制、导出或删除会话与材料。

## 6. UI/UX 核心原则

- 上传材料是首页主入口。
- 当前屏幕只处理一个主要任务。
- 当前概念和主推荐动作始终明确。
- 次要动作存在但视觉弱于主动作。
- 错误不是红叉式人格否定，而是可操作的信息。
- 用户可暂停、隐藏时间、返回来源和选择其他路径。
- 所有 AI 内容显示来源类型。
- 每个异步过程都显示进度、取消和恢复方式。
- 每个失败状态都保留用户已有进度。

## 7. 生产级要求

### 状态完整

实现加载、空、部分成功、解析失败、模型输出无效、网络失败、搜索失败、离线、保存冲突、权限拒绝和删除确认。

### 可访问

目标符合 WCAG 2.2 AA：键盘可操作、焦点可见、语义控件、非颜色单一表达、可访问对话框、足够目标尺寸和清楚错误关联。

### 响应式

桌面三栏、平板可折叠两栏、手机单栏和底部导航。移动端仍能完成上传、学习、Tutor、Quiz、复述、反馈、Agent 决策和恢复。

### 透明与可控

用户能查看来源、Agent 推荐理由、搜索原因、AI 活动记录、数据保存状态和删除范围。

### 数据隔离与保存

MVP 使用匿名浏览器 workspace 隔离数据，不实现复杂账户系统。原文件使用服务端 UUID 保存于非公开目录；数据库按 workspace 隔离会话和来源。复制、导出和删除范围遵循 `docs/18_DECISIONS_AND_SCOPE_CN.md`，删除全部数据不可恢复。

## 8. MVP 必须完成

- 上传并解析三种文件格式和粘贴文本
- 真实来源定位与来源查看器
- 知识地图和两分钟启动动作
- 专注学习工作区
- Contextual Tutor
- Quiz 与自由复述
- 分层提示
- 即时反馈与具体鼓励
- 错误驱动补救练习
- `LearningEvidence`
- 适应性 Agent
- 受控网络搜索
- 自动保存、恢复和总结
- Demo mode 与真实 GPT-5.6 模式
- 生产级异常状态、响应式与可访问基础

## 9. 明确暂不做

- 医疗诊断或治疗
- 多 Agent 群体架构
- 教师管理后台
- 社交排行榜或强制连续打卡
- 无限自动爬网
- 复杂账户付费体系
- 完整长期间隔重复系统

这些功能不得在截止前挤占核心闭环。
