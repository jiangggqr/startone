# StartOne 完整产品方案

## 1. 产品定义

StartOne 是一个帮助用户跨过启动困难、快速形成学习动量、建立知识体系并持续学到会的专注型学习 Web App。它以上传学习材料为核心，由适应性学习规划 Agent 驱动，并包含情境化 Tutor、可视化知识框架与完整掌握循环。

产品首发聚焦 AI 与技术学习，因为这类材料更新快、概念依赖强、术语密集，用户也更常从论文、课程讲义和技术文档开始。但当前 MVP 的学习机制是来源驱动而非学科专用，因此不把能力硬限制为只能学习 AI；有清晰文字材料的其他学科也可以使用。

它不是：

- 自动抓取大量链接的搜索产品
- 只有聊天框的通用 AI Tutor
- 只有番茄钟、待办或选择题的学习工具
- 声称诊断或治疗 ADHD 的医疗产品

它解决的核心问题是：

> 用户想学习并拥有材料，却难以快速开始、看清知识之间的结构、在注意力中断后继续推进，也无法确认自己是否真正理解并完成了学习。

### StartOne Momentum Loop

产品差异不只是“把材料变成讲解和题目”，而是把启动、专注、理解、记忆和连续完成做成一个自动衔接的行为循环：

1. **Start one**：上传后不填写目标、基础、时间或精力；AI 直接给出一个可在数分钟内完成的第一步。
2. **See the whole**：用可点击的概念节点和依赖连线显示完整知识框架，降低“不知道从哪里开始”的负担。
3. **Focus on one**：当前屏幕只保留一个概念，并按“大意—在框架中的位置—关键部分—具体例子—记忆锚点”解释。
4. **Retrieve once**：讲解后用 3 道定义 / 机制 / 应用题或 1 次自由复述立即提取，而不是继续被动重读。
5. **Win and continue**：即时反馈明确本次真实进步和唯一待修正点；安全的 Agent 动作自动执行并进入下一学习任务，不再停在额外确认页。只有材料确实不足时，系统才用一个明确提示请用户补充上传材料。

“快速开始”不等于跳过理解，“保持专注”不等于强制计时，“获得正反馈”不等于无依据的积分或夸奖。每个小胜利都必须对应真实完成或真实学习表现。

## 2. 目标用户

首要用户是使用课程讲义、论文、教材章节和笔记进行自学的大学生与成人学习者，尤其关注可能存在任务启动困难、工作记忆负担、时间感不清和错误反馈压力的用户。

产品使用 ADHD-informed 设计，但不要求用户声明诊断，也不把所有学习困难归因于 ADHD。

## 3. 内容来源边界

### 唯一学习来源：用户上传或粘贴的材料

PDF、Markdown、TXT 或粘贴文本。系统保留真实页码、标题和行号。StartOne 不搜索网络，也不从外部知识库自动补课。

AI 可以在已验证材料范围内重组结构、改写语言、提出问题和生成帮助理解的表述，但这些生成内容不是新的学习来源，不得引入材料没有支持的事实。所有讲解、例子、题目和反馈都必须能回到上传材料。

如果存在明确、命名并经过服务端验证的 `SourceGap`，系统不搜索；Agent 只能选择 `request_more_material`，保留当前概念并打开现有上传区，在其中说明缺少什么以及为什么需要，并提供“上传补充材料”或“继续当前范围”两个恢复动作，不增加独立确认页。

## 4. 五个系统

### 4.1 学习内容与来源系统

负责上传、解析、真实定位、材料内检索、来源预览、来源覆盖和缺口识别。

不负责教学、评分或决定下一步。

### 4.2 ADHD-informed 专注会话与 Momentum Loop

负责直接开始第一个概念、一屏一个概念、一个主动作、关系可视化、记忆锚点、可选时间显示、自动保存、暂停、恢复和下次启动动作。学习开始前不设自评、基线测试或强制任务；安全的下一学习动作由系统自动衔接，避免额外选择和确认成本。

不负责判断答案。

### 4.3 Guided Mastery Loop

统一负责当前概念的：

- Tutor 对话
- 连续引导
- Quiz
- 自由复述
- 分层提示
- 即时反馈
- 具体鼓励
- 错误驱动的补救练习
- `LearningEvidence` 生成

### 4.4 Adaptive Planning Agent

学习适应判断只读取经过验证的学习证据；同时读取路线、剩余会话时间、前置关系、来源覆盖和重试次数作为约束，只选择一个全局下一动作：

- `continue_next`
- `retry_current`
- `switch_activity`
- `simplify_current`
- `insert_prerequisite`
- `review_previous`
- `request_more_material`
- `finish_session`

Agent 不负责讲解、出题、评分或写鼓励文案。

Guided Mastery Loop 可以在 Feedback 中给出一个局部 `next_micro_action`，但它只能继续当前概念内的练习，不能改变路线、请求补充材料或结束会话。

### 4.5 平台、状态与信任层

负责 FastAPI、SQLite、GPT-5.6 Responses API、结构化输出、工具执行、状态机、权限、自动保存、内部确定性测试模式、错误恢复、测试和部署。测试模式及评审夹具不作为学习功能显示给用户。

## 5. 完整用户旅程

1. 首页以“上传材料并新建学习会话”为主入口，同时显示历史会话。
2. 用户上传材料；解析结果、成功文件和恢复操作直接显示在上传区域内。
3. 用户点击唯一主动作后，AI 分析材料覆盖，选择一个可解释的学习重点，并生成可视化知识框架、依赖关系和短路线。
4. 用户在带有节点与连接线的知识框架中看到概念依赖；每个概念可点击查看定义、作用和前置关系，再通过唯一主动作进入第一个概念。查看节点不改变当前学习路线，也无需填写目标、基础、时间或精力表单或完成开始前测试。
5. 进入专注工作区，一次学习一个概念，首先显示“大意—与前后概念的关系—关键部分—具体例子—一句记忆锚点”。系统不得仅凭材料声称知道用户的已有水平。
6. 来源作为讲解下方的轻量可展开引用，标明文件、位置和来源类型，不使用独占大屏的默认来源查看器。
7. 用户阅读讲解后，可以打开 Tutor 或“Test your understanding”；测试只提供两种清晰入口：3 道单选题或 1 道自由复述。Quiz 页面只保留三道题、选项、一个可选提示入口和提交按钮，不展开文件名、页码或系统边界。
8. Quiz 提交后显示总分，并逐题显示正确 / 需要复习、用户选择、正确答案（答错时）和简短原因；不把内部结构化字段、补救策略或技术术语逐项展示给学习者。
9. 学习活动在后台形成客观 `LearningEvidence`；它只记录表现，不包含下一步建议，也不需要独立的用户确认页面。
10. Agent 只根据已验证的 `LearningEvidence` 选择一个下一步，并用路线与剩余时间等信息做可行性约束；安全动作由系统自动执行并进入相应概念、练习、Tutor 或总结，不设置单独的下一步确认页。后台计时器不得在知识框架尚有已验证下一概念时提前结束会话。
11. 如果已验证的材料缺口阻断学习，`request_more_material` 保留当前概念并打开现有上传区的简短提示；用户可以上传相关材料或继续当前范围，不增加独立页面，系统绝不调用网络搜索。
12. 会话结束时保存总结、误区、进度和下次两分钟启动动作。
13. 用户可在学习记录中恢复、复制、导出或删除会话与材料。

## 6. UI/UX 核心原则

- 上传材料是首页主入口。
- 当前屏幕只处理一个主要任务。
- 从材料就绪到进入第一步只需要一个主动作；不要求学习者规划路线或选择支持强度。
- 当前概念和主推荐动作始终明确。
- 次要动作存在但视觉弱于主动作。
- 错误不是红叉式人格否定，而是可操作的信息。
- 每次完成后先显示与真实表现对应的小胜利，再自动衔接一个安全下一步；不使用虚假积分、强制连续打卡或无依据鼓励。
- 用户可暂停、返回来源和查看知识框架；会话时间、进度条和来源边界不占用独立侧栏。
- 来源类型在概念讲解和必要的补充内容处显示；Quiz 只保留一行弱化的来源类型说明，不重复文件与位置列表。
- `LearningEvidence`、评分字段、答案键和 Agent 边界属于后台机制，不作为默认学习界面内容。
- 每个异步过程都显示进度、取消和恢复方式。
- 每个失败状态都保留用户已有进度。

## 7. 生产级要求

### 状态完整

实现加载、空、部分成功、解析失败、模型输出无效、网络/API 失败、材料缺口、离线、保存冲突、权限拒绝和删除确认。

### 可访问

目标符合 WCAG 2.2 AA：键盘可操作、焦点可见、语义控件、非颜色单一表达、可访问对话框、足够目标尺寸和清楚错误关联。

### 响应式

桌面使用知识框架与当前内容两栏，平板可折叠地图，手机使用单栏和底部导航。移动端仍能完成上传、学习、Tutor、Quiz、复述、反馈、Agent 决策和恢复。

### 透明与可控

用户能查看材料来源、Agent 推荐理由、材料缺口说明、AI 活动记录、数据保存状态和删除范围。

### 数据隔离与保存

MVP 使用匿名浏览器 workspace 隔离数据，不实现复杂账户系统。原文件使用服务端 UUID 保存于非公开目录；数据库按 workspace 隔离会话和来源。复制、导出和删除范围遵循 `docs/18_DECISIONS_AND_SCOPE_CN.md`，删除全部数据不可恢复。

## 8. MVP 必须完成

- 上传并解析三种文件格式和粘贴文本
- 真实来源定位与轻量内联引用
- 知识地图和直接开始的第一个概念讲解
- 专注学习工作区
- Contextual Tutor
- Quiz 与自由复述
- 分层提示
- 即时反馈与具体鼓励
- 错误驱动补救练习
- `LearningEvidence`
- 适应性 Agent
- 明确的材料缺口提示与补充上传恢复路径
- 自动保存、恢复和总结
- 内部确定性测试路径与真实 GPT-5.6 模式严格隔离；学习界面不显示测试模式或评审入口
- 生产级异常状态、响应式与可访问基础

## 9. 明确暂不做

- 医疗诊断或治疗
- 多 Agent 群体架构
- 教师管理后台
- 社交排行榜或强制连续打卡
- 无限自动爬网
- 无材料的“只输入主题”生成路径
- 强制用户填写学习目标、已有基础、时间或精力的前置表单
- 复杂账户付费体系
- 完整长期间隔重复系统

这些功能不得在截止前挤占核心闭环。
