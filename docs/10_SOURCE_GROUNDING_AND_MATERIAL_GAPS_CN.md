# 10 来源定位、材料覆盖与补充材料缺口

## 1. 上传格式

- PDF
- Markdown
- TXT
- 粘贴文本

文件限制应由配置控制，并在上传前显示。扫描 PDF 无文字层时应提示用户更换文件或后续增加 OCR，不得伪造内容。

## 2. 稳定定位

### PDF

保存：文件 ID、页码、页内片段顺序、字符范围和片段文本。

### Markdown / TXT

保存：文件 ID、标题路径、开始行、结束行、字符范围和片段文本。

### 粘贴文本

保存：文档标题、段落编号和字符范围。

每个 `source_ref` 必须能返回数据库中的真实片段。

## 3. 切分与检索

- 先按标题、页和段落结构切分
- 再按长度细分
- 保留重叠但避免改变位置
- 使用 SQLite FTS5 或等效本地检索
- 模型调用优先发送相关片段，不发送不必要的整本材料

## 4. 来源类型

```text
uploaded
```

上传文件和粘贴文本都是 `uploaded`。所有用户可见的概念、解释、题目、反馈和例子都必须回到已验证的上传来源。

AI 可以对材料做结构化、简化和改写，界面用“Generated from your material”说明生成性质，但它不构成第二学习来源，也不能引入材料未支持的事实。

## 5. 来源覆盖与缺口

`SourceGap` 至少包含：

```text
gap_id
concept_id
description
why_needed
evidence
current_source_refs
requested_material_scope
```

缺口必须具体，例如“材料没有解释矩阵乘法怎样影响 Q/K/V”，不能只写“资料不足”。

## 6. 材料缺口恢复

当命名 `SourceGap` 经过服务端验证且确实阻断学习时，Agent 可以选择 `request_more_material`。系统必须：

- 用学习者能理解的一句话说明缺少什么以及为什么需要
- 提供上传 PDF、Markdown、TXT 或粘贴文本的主动作
- 提供继续当前材料范围的次动作，并保留进度
- 新材料解析完成后重新验证覆盖并回到原概念

系统不推荐网站、不展示链接候选、不调用互联网；Tutor、地图生成、练习、反馈和 Agent 调用都不获得任何网络工具。

## 7. 补充材料失败

- 不丢失当前会话
- 逐文件显示解析失败和恢复动作
- 允许“继续当前材料范围”
- 保留命名缺口供用户以后补充

## 8. 防止来源幻觉

- 模型输出的所有 `source_ref` 必须服务端验证
- 未验证引用不得渲染
- 引用位置错误可以由用户报告
- AI 生成的改写不能生成虚假页码、行号或材料外事实
