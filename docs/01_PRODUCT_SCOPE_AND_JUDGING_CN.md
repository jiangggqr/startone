# 01 产品范围、差异化与评分映射

## 核心问题

现有 AI Tutor 通常假设用户已经开始，并要求用户自己提出问题、选择资料、判断先后顺序和决定何时练习。StartOne 解决的是更完整的问题：从“想学习但难启动”，到建立知识体系、一次掌握一个概念、在中断后恢复，再到基于证据确认学习是否完成。

## 学科定位

- 首发市场与演示内容聚焦 AI 与技术学习：材料密集、前置关系明显、更新快，最能体现来源核查、知识地图与适应性推进的价值。
- 当前 MVP 不宣称“只学习 AI”。解析、Tutor、Quiz、复述、证据与 Agent 路线机制本身适用于有清晰文字材料的多种学科。
- 只有在加入代码执行与调试、公式和架构图理解、AI 专属先修知识库、技术版本时效检查和 AI 学习专项评测后，才应把产品定义为 AI-only。
- 因此当前对外定位采用 **AI-first, not AI-only**。

## 真实受众

- 有课程 PDF、讲义、论文或笔记的大学生与成人学习者
- 需要快速理解复杂技术或理论材料的人
- 可能存在任务启动困难、工作记忆负担、时间感不清或错误反馈压力的用户

## 差异化

1. 上传材料优先，而不是默认生成通用课程。
2. 每个知识点和练习可以回到真实来源位置。
3. Tutor 负责当前概念；Agent 只负责下一步，职责清楚。
4. 学习证据不仅包括 Quiz 分数，还包括复述覆盖、提示深度、Tutor 困惑信号、时间与来源缺口。
5. ADHD-informed 机制进入流程和界面，而不是只写在宣传文案中。

## 对竞赛评分的映射

### Technological Implementation

- GPT-5.6 Responses API
- Structured Outputs
- 受控 function/tool calling
- PDF/Markdown/TXT 来源定位
- SQLite 状态与恢复
- LearningEvidence 与 Agent 状态机
- 受控 web search
- Demo/real 双模式与评测

### Design

- 完整页面、交互、异常状态和移动端流程
- 可点击低保真原型作为实现依据
- 一个主动作、可恢复进度、来源透明和可控搜索

### Potential Impact

解决“难以开始、缺少知识结构、注意力中断后难以推进，以及无法判断是否真正学会”的具体问题；可测量启动完成率、概念完成率、提示深度、误区修复和会话恢复。

### Quality of the Idea

产品单位不是聊天消息或计时器，而是：

> grounded initiation-to-mastery loop

## MVP 边界

必须完成上传到总结的完整端到端流程。不得为了展示复杂性加入多 Agent、教师后台、社交排行榜或无限搜索。
