# 09 Adaptive Planning Agent

## 核心职责

Agent 回答一个问题：

> 基于当前学习证据和剩余条件，下一步做什么？

## 输入

- 当前概念与路线
- 经过 Schema、来源和状态验证的 `LearningEvidence` 列表（唯一学习表现依据）
- 从 Evidence 聚合得到的当前掌握状态
- 剩余会话时间
- 前置知识关系
- 来源覆盖和命名缺口
- 用户搜索权限
- 已发生的重试次数
- 用户搜索偏好与当前暂停状态

除 Evidence 外的输入只用于约束动作是否可行。例如剩余时间可以让系统选择更短的练习，但不能在知识框架还有已验证下一概念时强制结束，也不能在没有学习证据时宣称用户已经掌握或存在误区。

## 动作枚举

```text
continue_next
retry_current
switch_activity
simplify_current
insert_prerequisite
review_previous
request_search
finish_session
```

每次只能返回一个动作。

## 决策输出

```text
action
reason_for_user
estimated_minutes
target_concept_id
return_to_concept_id
required_tool
confidence
```

`reason_for_user` 必须简短、可理解，不暴露隐藏推理。

## 关键规则

### 继续

只有当核心要点达到最低掌握阈值，且没有阻断性误解时才推荐。

### 重试或换练习

一次局部错误优先使用重试或换练习，不立即插入大段前置课程。

### 插入前置知识

需要同时满足：

- 当前困难与前置概念有明确关系
- 至少一个证据支持该判断
- 插入任务较短
- 保存返回原概念的路径

### 请求搜索

需要：

- 命名来源缺口
- 上传材料无法覆盖
- 用户允许建议搜索
- 本地材料和现有 AI 简化仍不足

Agent 只能提出 `request_search`。真正调用搜索前必须再次由用户确认。

### 结束会话

自动流程只有在计划路线全部掌握，或循环保护判定已经没有任何有效学习动作时才能使用；后台计时器不得提前结束未完成路线。用户随时可以主动暂停并保存，不需要由 Agent 把暂停解释为失败。结束时必须生成下次启动动作。

## 用户控制

生产学习界面不显示普通 Agent 决策页。用户从反馈选择 Keep going 后，系统自动应用唯一的安全动作并进入相应概念、练习、Tutor 或总结；`request_search` 只能进入独立搜索确认。用户仍可暂停、返回知识框架或退出会话；这些普通导航不被视为学习错误。服务端保留受验证的覆盖接口用于恢复和兼容，但不作为默认学习任务展示。

## 循环保护

- 同一概念最多连续两次相同补救策略
- 同一 Agent 动作不得无条件重复
- 插入前置知识后必须返回原概念或由用户结束
- 搜索失败后不得重复自动搜索
- 无有效动作时安全结束并保存进度

## 工具边界

Agent 通过受控服务器工具改变状态。它不能直接写数据库、生成 HTML、发送任意 URL 或调用未注册工具。
