# 02 信息架构与导航

## 全局导航

- 首页 / 资料库
- 学习记录
- 设置
- 使用说明

学习会话中额外显示：

- 面包屑：会话名 › 当前概念
- 自动保存状态
- 暂停与退出

## 页面层级

### 会话前

1. 首页与资料库（上传材料并新建会话是唯一主操作）
2. 上传与解析
3. AI 材料分析与学习重点生成
4. 简洁知识框架与开始学习

### 会话中

6. 专注学习工作区（首先讲解）
7. Tutor
8. Quiz
9. 自由复述与提示
10. 简短即时反馈
11. 补救练习
12. 简洁下一步（Evidence 与 Agent 判断不单独展示）
13. 受控搜索确认
14. 外部来源选择
15. 讲解内的轻量来源引用

### 会话后与管理

17. 会话总结与恢复
18. 学习记录
19. 设置、隐私与数据控制
20. 跨页面加载、空、错误与恢复状态
21. 响应式和键盘行为

## 建议路由语义

- `/`
- `/sessions/{id}/sources`
- `/sessions/{id}/path`
- `/sessions/{id}/learn/{concept_id}`
- `/sessions/{id}/summary`
- `/history`
- `/settings`

Tutor、Quiz、复述和反馈可以使用嵌套路由或明确的抽屉/全屏状态。来源默认作为讲解内可展开的引用。浏览器刷新必须恢复相同会话、概念、活动和未提交输入。

首页只有材料驱动入口。至少一份材料可读后，用户用一个主动作触发覆盖分析、AI 学习重点和知识地图生成；不出现目标、基础、时间或精力设置关卡。来源覆盖仍可从学习地图返回审阅，但不是另一条入口。

## 桌面布局

专注工作区使用两栏：

- 左：可点击的可视化知识框架；点击节点查看定义、作用和依赖，不直接改变当前路线
- 中：当前概念和主动作

保存与暂停放在顶部轻量工具栏；不设置独立 Session status 侧栏。Tutor 可作为抽屉或独立状态。Quiz 与复述使用主区域；Quiz 不显示文件与页码列表、系统边界或锁定提示卡。提交后的反馈也使用单列居中布局。

## 移动端布局

- 单栏内容
- 底部导航：地图、学习、Tutor、更多
- Quiz、复述和反馈使用全屏；反馈只显示结果、答案说明和 Continue
- 主动作固定在内容后，不遮挡文本
- 不保留桌面固定侧栏
