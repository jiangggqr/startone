# 12 无障碍与响应式规范

## 标准目标

以 WCAG 2.2 AA 为目标，并参考 WAI-ARIA Authoring Practices 的常见组件模式。

## 键盘

- 所有核心流程可只用键盘完成
- `Tab` 顺序符合视觉和阅读顺序
- Enter/Space 正确触发按钮、单选和复选控件
- Esc 关闭可关闭的抽屉或模态框
- 模态框打开时焦点进入内部并被约束
- 关闭后焦点返回触发按钮
- 页面状态切换后焦点移动到新页面主标题或错误摘要
- 不创建键盘陷阱

## 焦点

- `:focus-visible` 清晰可见
- 不仅改变细微颜色
- 固定头部不能遮挡焦点元素
- 错误提交后焦点移动到错误摘要或第一个错误字段

## 语义

- 使用真实 button、input、label、fieldset、legend、nav、main、aside、dialog
- Quiz 的每一道单选题使用独立 radio group，并以题目作为 fieldset legend
- Tabs 遵循 tab / tabpanel 语义
- 展开提示使用 disclosure 语义
- 保存状态和异步结果使用适当 live region，但避免重复朗读流式每个 token

## 视觉与理解

- 正确、错误、当前和锁定状态不能只依赖颜色
- 正文支持放大到 200% 而不丢功能
- 文字和背景达到 WCAG 对比要求
- 语言简洁，避免一次给出多项复合指令

## 点击目标

WCAG 2.2 AA 的最小目标尺寸要求为 24×24 CSS px（有例外）。本产品主按钮、常用工具和移动端触控目标默认至少 44×44 CSS px。

## 减少动画

遵循 `prefers-reduced-motion`。倒计时、进度和页面切换不使用无法关闭的跳动或闪烁效果。

## 响应式断点

### ≥1200px

两栏专注工作区：可点击知识框架 / 当前概念。保存与暂停位于顶部，不设置 Session status 侧栏。

### 768–1199px

- 地图可折叠
- 当前概念为主栏
- 保存与暂停放入顶部
- Tutor 使用抽屉

### <768px

- 单栏
- 底部导航
- Tutor、Quiz、复述和反馈全屏；来源引用在概念讲解内就地展开
- 不使用横向滚动展示关键内容
- 主按钮位于自然阅读顺序中
- 生产页面使用真实底部导航完成地图、学习、Tutor 和更多之间的切换；不能只展示静态手机示意图

## 屏幕阅读器文案

图标必须有可访问名称。进度表达使用“已完成 2/5 个概念”，不能只朗读“40%”。
