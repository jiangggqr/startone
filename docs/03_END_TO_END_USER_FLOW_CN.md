# 03 完整用户流程与分支

## 主流程

1. 用户从首页主入口选择“上传材料并新建会话”。
2. 用户上传材料；系统在上传区域内逐文件显示解析进度和结果。
3. 用户处理失败文件或继续使用成功文件。
4. 用户点击“Build my learning path”。
5. GPT-5.6 从材料中识别覆盖、选择一个学习重点，并生成知识框架、依赖和本次路线；它不推断用户已有水平。
6. 用户查看按依赖顺序排列的可视化知识框架；每个节点都可点击查看定义、作用和前置关系，但不会因此跳过学习路线。
7. 用户点击“Start learning the first concept”后直接进入第一个概念；无设置表单，无开始前测试。
8. 页面先显示概念定义、关键步骤、具体例子、在知识框架中的作用，以及可展开的轻量来源引用。
9. 用户阅读后选择 Tutor，或进入只有“3 道选择题 / 1 道自由复述”两种入口的理解测试；提示只在用户点击后原位出现。
10. Quiz 提交后显示总分，并逐题显示正确 / 需要复习、答案核对和简短原因，然后只给一个 Continue。
11. 活动在后台规范化为只含观察事实的 `LearningEvidence`，不设独立审阅页。
12. 用户点击 Continue 后，Agent 只基于已验证证据选择一个下一动作。
13. 界面只显示该动作、一句理由和 Continue，不再额外展示替代路径表单。
14. 到达会话上限或当前路线完成后，系统生成总结和下次启动动作。
15. 用户退出后可从学习记录恢复。

## Tutor 分支

- 用户点击快捷问题或自由输入。
- Tutor 先澄清具体困难。
- 逐级提供方向、结构、关键词、部分例子和简练解释。
- Tutor 提出检查问题。
- 对话产生困惑标签、支持层级和前置知识信号。
- Tutor 不直接改变路线。

## Quiz 分支

- 每个概念的 Quiz 固定包含 3 道单选题，分别覆盖定义、机制和应用；页面显示三题、选项、一行弱化来源类型和提交按钮。
- 用户可按需展开分层提示，不显示未解锁卡或成功横幅。
- 提交后显示总分，并逐题显示结果、所选答案、正确答案（答错时）和选项级解释。
- 记录正确性、误解标签、提示深度和时间。

## 自由复述分支

- 用户在不看讲解的情况下回答。
- 提示逐级揭示。
- 系统评估要点覆盖、错误和表达模糊处。
- 反馈不要求与参考答案逐字一致。

## 补救分支

- 根据最新具体缺口，提供更简单解释、更小问题或具体例子。
- 连续两次无改善时停止重复同一形式。
- 形成新的证据后交给 Agent。

## 外部搜索分支

只有四重条件满足才进入：

1. 会话允许建议搜索。
2. 存在命名的来源缺口。
3. Agent 选择 `request_search`。
4. 用户在看到缺口和搜索范围后确认本次搜索。

执行端点必须重新验证四项条件；失败后返回原学习状态，不丢进度。

## 恢复分支

刷新、关闭页面、离线或 API 失败后：

- 恢复当前会话、概念、活动和未提交文本。
- 显示最后同步时间。
- 冲突时优先保护用户输入，并要求明确选择版本。
