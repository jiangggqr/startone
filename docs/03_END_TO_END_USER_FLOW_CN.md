# 03 完整用户流程与分支

## 主流程

1. 用户从首页主入口选择“上传材料并新建会话”；也可先创建会话设置，再在下一步上传。
2. 设置目标、基础、时间、精力、语言和支持偏好。
3. 上传材料；系统逐文件显示解析进度。
4. 用户处理失败文件或继续使用成功文件。
5. 系统显示来源结构、位置和候选知识缺口。
6. GPT-5.6 生成知识地图、依赖和本次路线。
7. 用户确认路线或调整时间/重点。
8. 系统显示一个 60–120 秒启动动作。
9. 用户完成动作并进入当前概念。
10. 用户选择 Tutor、Quiz、复述、提示或来源查看器。
11. 系统产生即时反馈和必要补救练习。
12. 活动被规范化为 `LearningEvidence`。
13. Agent 选择一个下一动作并显示理由。
14. 用户接受推荐或选择其他路径。
15. 到达时间上限或目标后，系统生成总结和下次启动动作。
16. 用户退出后可从学习记录恢复。

## Tutor 分支

- 用户点击快捷问题或自由输入。
- Tutor 先澄清具体困难。
- 逐级提供方向、结构、关键词、部分例子和简练解释。
- Tutor 提出检查问题。
- 对话产生困惑标签、支持层级和前置知识信号。
- Tutor 不直接改变路线。

## Quiz 分支

- 系统显示一题和来源标签。
- 用户可使用分层提示。
- 提交后立即显示选项级解释。
- 记录正确性、误解标签、提示深度和时间。

## 自由复述分支

- 用户在不看讲解的情况下回答。
- 提示逐级揭示。
- 系统评估要点覆盖、错误和表达模糊处。
- 反馈不要求与参考答案逐字一致。

## 补救分支

- 根据最新具体缺口，提供更简单解释、更小问题或具体例子。
- 连续两次无改善时停止重复同一形式。
- 形成新的证据后交给 Agent。

## 外部搜索分支

只有四重条件满足才进入：

1. 会话允许建议搜索。
2. 存在命名的来源缺口。
3. Agent 选择 `request_search`。
4. 用户在看到缺口和搜索范围后确认本次搜索。

执行端点必须重新验证四项条件；失败后返回原学习状态，不丢进度。

## 恢复分支

刷新、关闭页面、离线或 API 失败后：

- 恢复当前会话、概念、活动和未提交文本。
- 显示最后同步时间。
- 冲突时优先保护用户输入，并要求明确选择版本。
