# 10 来源定位与受控搜索

## 1. 上传格式

- PDF
- Markdown
- TXT
- 粘贴文本

文件限制应由配置控制，并在上传前显示。扫描 PDF 无文字层时应提示用户更换文件或后续增加 OCR，不得伪造内容。

## 2. 稳定定位

### PDF

保存：文件 ID、页码、页内片段顺序、字符范围和片段文本。

### Markdown / TXT

保存：文件 ID、标题路径、开始行、结束行、字符范围和片段文本。

### 粘贴文本

保存：文档标题、段落编号和字符范围。

每个 `source_ref` 必须能返回数据库中的真实片段。

## 3. 切分与检索

- 先按标题、页和段落结构切分
- 再按长度细分
- 保留重叠但避免改变位置
- 使用 SQLite FTS5 或等效本地检索
- 模型调用优先发送相关片段，不发送不必要的整本材料

## 4. 来源类型

```text
uploaded
external
ai_supplement
```

所有用户可见的概念、解释、题目、反馈和例子都必须带来源类型。

`pasted` 是材料介质，不是第四种 UI 来源；粘贴文本在界面中显示为 `uploaded`。

## 5. 来源覆盖与缺口

`SourceGap` 至少包含：

```text
gap_id
concept_id
description
why_needed
evidence
current_source_refs
suggested_query_scope
```

缺口必须具体，例如“材料没有解释矩阵乘法怎样影响 Q/K/V”，不能只写“资料不足”。

## 6. 受控搜索

四重搜索门：

1. `search_permission = true`
2. 存在服务端验证的命名 `SourceGap`
3. Agent 动作是 `request_search`
4. 用户确认本次搜索的目标与范围

服务端只在执行端点重新验证四项条件后提供 `web_search` 工具，并要求本次调用执行搜索。模型不能在 Tutor、地图生成或普通 Agent 决策调用中看到搜索工具。

搜索结果只保留少量候选，每项显示：

- 标题
- URL
- 发布者或机构
- 选用原因
- 填补的具体缺口
- 访问时间
- 引用片段

## 7. 搜索失败

- 不丢失当前会话
- 不自动无限重试
- 提供“基于现有材料继续”
- 保留缺口供以后处理
- 外部来源打不开时不作为事实依据

## 8. 防止来源幻觉

- 模型输出的所有 `source_ref` 必须服务端验证
- 未验证引用不得渲染
- 引用位置错误可以由用户报告
- AI 补充解释不能生成虚假页码或行号
