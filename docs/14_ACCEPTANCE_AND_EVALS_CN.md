# 14 测试、评测与验收

## 1. 验收层级

### 单元测试

- 文件解析和真实位置
- 稳定 ID 与校验和
- 状态转换
- Schema 验证
- 来源引用验证
- Agent 动作枚举和循环保护
- 权限门
- workspace 数据隔离、乐观版本冲突和删除级联

### 集成测试

- 上传 → 覆盖 → 地图
- Tutor 消息持久化
- Quiz/复述 → 反馈 → Evidence
- Evidence → Agent 决策 → 状态执行
- 搜索确认 → 结果选择 → 插入前置任务
- 自动保存和刷新恢复
- 暂停覆盖状态 → 恢复原业务状态
- 会话复制共享不可变 blob、删除最后关联后清理 blob

### 浏览器流程测试

- 桌面主流程
- 移动端主流程
- 仅键盘主流程
- 失败文件部分成功
- 模型输出无效恢复
- 搜索失败返回当前会话
- 删除确认和焦点返回
- 保存冲突版本选择、权限拒绝和来源引用无效恢复

### 模型评测

使用固定材料和用户回答评估：

- 知识地图是否忠于来源
- Quiz 干扰项是否对应误解
- 复述反馈是否接受合理改写
- 鼓励是否具体而非泛化
- Tutor 是否保持在当前概念
- Agent 是否只选择一个动作
- 无缺口时是否拒绝搜索

## 2. 核心端到端场景

### 场景 A：正常推进

用户上传 Transformer 材料，完成 Quiz 与复述，Evidence 达标，Agent 推荐继续下一概念。

### 场景 B：补救后推进

用户知道“相关程度”但不清楚“加权汇总”；系统给补救练习；改善后 Agent 继续。

### 场景 C：插入前置知识

用户连续在 Q/K/V 中暴露矩阵乘法缺口；Agent 插入短前置概念；完成后返回原概念。

### 场景 D：受控搜索

材料确实缺少前置解释，会话允许建议搜索，存在验证后的命名缺口，Agent 请求；用户确认；系统选择少量来源并明确标注。任一条件缺失时不得调用网络。

### 场景 E：拒绝搜索

用户拒绝；会话不丢失；系统基于现有材料继续或结束。

### 场景 F：恢复

用户在复述中关闭页面；重新打开后输入、提示深度、当前概念和剩余时间恢复。

## 3. UX 验收

- 每屏只有一个视觉最强主动作
- 当前概念始终可见
- 时间可隐藏
- 加载、空、错误和部分成功均有恢复动作
- 来源类型持续可见
- Agent 理由可见且可覆盖
- 移动端不出现关键横向滚动
- 主流程只用键盘可完成
- 页面切换后焦点进入新标题；对话框关闭后焦点返回触发控件

## 4. 不可接受的“通过”方式

- 硬编码固定答案冒充真实模式
- 关闭失败测试
- 捕获所有异常后静默继续
- 使用不存在的页码
- 搜索权限关闭时仍调用网络
- 只有截图，没有可运行交互
- 只测试 API，不打开浏览器
- 只显示“正确/错误”，没有解释

## 5. 最终完成标准

- `evals/` 中全部案例有自动化或明确人工验证记录
- Demo mode 从干净环境可运行
- 真实 GPT-5.6 至少完成一次烟雾测试
- 浏览器无未处理错误
- 不含密钥、缓存、数据库副本和私有材料
- README 能让评委独立完成主流程
