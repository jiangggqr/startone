# 11 生产级 UX 标准

## 1. 状态完整性

每个异步操作实现：

- idle
- loading
- streaming（适用时）
- success
- partial_success
- error
- cancelled
- retrying

不得只实现成功路径。

## 2. 加载与等待

- 小于约 500ms：无需闪烁式加载器
- 超过约 1s：显示骨架或明确状态
- 长任务显示阶段，例如“提取第 12/48 页”
- 超过 10s 提供取消或后台继续
- GPT 输出可使用流式文本，但结构化对象最终必须完整验证

## 3. 自动保存与恢复

- 用户输入变化后自动保存草稿
- 明确显示正在保存、已保存、离线保存和失败
- 刷新恢复当前会话、概念、活动和未提交文本
- 服务端版本冲突时保护用户输入，不能静默覆盖
- 使用乐观版本号；冲突界面同时保留本地和服务端版本并提供明确选择
- 暂停保存 `resume_state`，恢复到原概念、活动、草稿、提示深度和计时状态

## 4. 错误信息

错误文案包含：

1. 发生了什么
2. 哪些内容已经保存
3. 用户现在能做什么
4. 技术详情入口（仅需要时）

普通用户界面不直接显示堆栈、JSON Schema 或内部调用 ID。

## 5. 部分成功

多文件、多个来源或多个模型子任务中，成功部分应可继续使用。失败项独立重试或移除。

## 6. Agent 可解释且不增加选择负担

- 保存推荐动作、简短理由和预计时间，供恢复与透明记录查看
- 普通安全动作自动应用，不增加一个确认页面
- 不显示隐藏推理
- 不显示替代路径列表或覆盖理由输入框
- `request_more_material` 保留当前概念并打开现有上传区，不创建独立确认页
- 用户暂停、退出或返回知识框架不受惩罚

## 7. AI 透明度

- 显示来源类型
- 明确说明 AI 生成的讲解来自上传材料
- 模型失败时不使用硬编码假结果冒充真实结果
- 确定性 fixture 只用于自动化测试和内部验收，不在最终学习界面提供模式或评审控制

## 8. 隐私与数据控制

- API Key 只在服务端
- 上传材料不写入公开日志
- 明确保存位置和保留策略
- 用户可删除单个文件、会话或全部数据
- 破坏性操作二次确认并说明范围
- 下载或导出用户学习记录
- MVP 通过安全的匿名 workspace Cookie 隔离数据，不接受请求体传入 workspace ID
- 原始上传文件使用服务端 UUID 名称保存在非公开目录

## 9. 安全

- 校验文件类型和大小
- 清理文件名和渲染内容
- 不把模型输出直接作为未转义 HTML
- 所有状态改变由服务端验证
- Agent 工具使用枚举和参数 Schema
- 将上传材料视为不可信内容，在模型指令中明确其内容不能覆盖系统规则或授权工具
- 学习流程不抓取外部 URL，不向模型暴露网络工具

## 10. 性能目标

这些是产品目标，不是竞赛硬性 SLA：

- 页面首次可交互尽量低于 2 秒（本地 Demo）
- 常规保存反馈低于 500ms
- Tutor 首个流式内容尽量低于 2 秒
- 长解析过程持续显示进度
- 移动端避免加载整份 PDF 预览

## 11. 事件与可观测性

至少记录：

- session_created
- source_parse_started/completed/failed
- path_confirmed
- start_action_completed
- tutor_turn
- activity_submitted
- hint_revealed
- feedback_shown
- remedial_completed
- agent_decision
- additional_material_requested/uploaded/declined
- session_resumed/ended

事件不得包含不必要的原始学习材料或完整自由文本。
